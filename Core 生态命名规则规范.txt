一、概述
1.1 定义
Core生态命名规则规范（Core Naming Convention Specification）定义了Core生态内所有硬件模块、组件和产品的标准化命名体系。本规范确保Core生态内所有产品线的命名具有一致性和可识别性。

1.2 适用范围
本规范适用于Core生态内所有产品、模块和组件的命名，包括但不限于：

中央处理器（CPU）

图形处理器（GPU）

主板（MOBO）

内存模块（RAM）

存储模块（NAND Flash）

电源模块（UPS）

通信模块（IC）

显示模块（Display）

音频模块（LoudSpeaker）

散热模块（CoolingModule）

影像模块（CameraModule）

安全模块（SecurityModule）

脑机接口（Brain）

二、命名格式
2.1 统一格式
Core生态所有硬件模块的命名遵循以下格式：

text
Core [代际] [类别缩写] [级别]
2.2 格式说明
组成部分	说明	示例
Core	生态品牌前缀，固定不变	Core
[代际]	数字表示产品世代	7、8、9
[类别缩写]	两位/三位字母表示模块类型	Gen、Cen、Wen
[级别]	表示产品定位（可选）	Mobile、Pro、Ultra、Max、Server

四、级别后缀定义
模块名称后可附加级别后缀，以明确产品定位：

后缀	含义	定位
Mobile	移动版	手机、平板、轻薄本
Pro	专业版	工作站、专业应用
Ultra	极致版	旗舰产品、顶级性能
Max	最大版	极致性能、最高规格
Server	服务器版	数据中心、云计算
Edge	边缘版	边缘计算、工业控制
（无后缀）	标准版	主流桌面、通用场景

六、命名规范要点
品牌前缀统一：所有产品均以“Core”开头，确保品牌识别的一致性。

代际数字递增：代际数字随产品世代递增（7、8、9……），不跳号、不重复。

类别缩写固定：每个模块类别拥有唯一的缩写标识，一经定义不得更改。

级别后缀可选：级别后缀仅在产品需要明确市场定位时使用，标准版可不加后缀。

大小写规范：“Core”首字母大写，“Gen”“Cen”等类别缩写首字母大写，代际数字与类别缩写之间空格分隔。
