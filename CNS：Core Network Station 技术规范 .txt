一、概述
1.1 定义
CNS（Core Network Station）是Core生态的算力调度中枢，负责将Core生态中所有设备的算力资源抽象为统一的算力池，并按需分配给任意接入设备。CNS既是一个硬件平台（固定节点），也是一套服务协议（算力调度与任务管理）。

1.2 核心定位
属性	说明
全称	Core Network Station
简称	CNS
类型	硬件 + 服务协议
硬件载体	CAPU / CCPU，功耗 ≥100W，7×24常驻
服务范围	Phone、Watch、Glass、Brain、TV、Book、Board 等所有Core设备
职责	算力注册、任务调度、状态管理、跨设备任务迁移
1.3 核心理念
CNS让算力与设备形态解耦。任何Core设备都可以通过CNS调用算力池中的资源，就像它们本身拥有这种能力一样。算力不再是特定设备的附属品，而是Core生态的公共资源。

二、CNS 的组成
CNS由硬件平台和服务协议两部分组成：

2.1 硬件平台
CNS硬件是固定部署的算力调度节点：

参数	规格
处理器	CAPU 或 CCPU 系列
功耗	≥100W（恒定运行）
运行状态	7×24常驻
形态	桌面式 / 机架式
网络	CLS光信号收发器 + 毫米波 + 有线以太网
存储	ECCHBM（任务状态缓存）
散热	主动风冷 / 被动散热（视部署环境）
2.2 服务协议
CNS服务协议定义了一套标准的通信规则：

服务	说明
算力注册	CNBP等算力提供者向CNS注册自身算力（CPU/GPU/NPU规格、当前负载、可用性）
任务请求	客户端设备向CNS发起算力请求（任务类型、优先级、数据位置）
任务调度	CNS根据算力池状态，分配任务到最优节点执行
状态同步	CNS维护任务状态，支持跨设备查询和恢复
任务迁移	设备切换时，任务状态由CNS迁移到新的客户端
三、CNS 的覆盖范围
3.1 算力消费者（发起任务）
设备	典型场景	对CNS的依赖程度
CorePhone	AI推理、渲染任务	高（本地算力不足时）
CoreWatch	生物数据分析	高（本地无算力）
CoreGlass	实时空间计算	极高（低延迟要求）
CoreBrain	神经信号解码	极高（实时双向流）
CoreTV	媒体转码、推荐计算	中（后台任务）
CoreBook	编译、渲染	中（可本地也可远程）
3.2 算力提供者（执行任务）
设备	算力规格	对CNS的角色
CNBP（WorkStation）	128核 + 10240GPU + 10240TOPS NPU	主要算力来源
CNBP（移动状态）	受限算力	轻量任务执行者
CoreTV（闲置时）	家庭SoC	轻量算力贡献者
CorePhone（插电+闲置）	移动SoC	轻量算力贡献者
四、CNS 的调度策略
CNS根据任务类型和设备特性，自动选择最优调度策略：

设备类型	调度策略	延迟要求	优先级
CoreGlass	就近调度（优先本地边缘节点）	<20ms	实时性优先
CoreBrain	专用通道，实时双向流	<10ms	确定性延迟优先
CorePhone	完整任务卸载到CNBP	<100ms	交互响应优先
CoreWatch	数据压缩上传，结果精简回传	<1s	低功耗优先
CoreTV	后台批量处理	数秒至数分钟	资源利用优先
五、跨设备任务迁移（状态延续）
CNS是Core生态“状态延续”理念在算力层面的实现：

用户在CorePhone上发起任务

CNS调度到CNBP执行

用户离开手机，拿起CoreGlass

任务状态通过CNS同步到CoreGlass

用户在CoreGlass上继续查看或控制任务

设备切换对用户透明，算力执行位置对用户透明。

六、部署场景
6.1 家庭算力网络
1台CNS（固定部署）

1台CNBP（算力提供者）

多个Core设备（Phone、Watch、TV、Glass）

CNS管理家庭内部的算力调度，所有设备共享CNBP的计算能力。

6.2 工作室算力集群
1台CNS（固定部署）

3-5台CNBP（算力提供者）

多个Core设备

CNS在多台CNBP之间分配任务，实现算力池化。

6.3 企业算力基础设施
多台CNS（分布式部署）

多台CNBP

多个CLSDN存储网络

CNS之间协同工作，形成跨区域的算力网络。

八、与现有组件的协同
组件	与CNS的关系
CLSDN	CNS运行在CLSDN网络之上，使用其互联能力
CLS	CNS通过CLS与CNBP通信，获取算力状态、下发任务
WLCLS / WLHBM	CNS的任务状态可持久化到存储池中，支持跨设备恢复
CNBP	CNS的主要算力来源，CNBP向CNS注册自身算力
CorePhone/Watch/Glass/Brain	CNS的客户端，发起任务请求并接收结果
