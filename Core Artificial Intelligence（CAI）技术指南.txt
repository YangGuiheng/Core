三、训练层
3.1 CAI训练中心（超大规模）
CAI训练中心是Core生态中超大规模AI模型的训练基础设施，负责从零训练或大规模预训练所有>10B参数的CAI模型。

属性	说明
硬件底座	CNBP集群 + 专用AI加速卡
训练框架	CAI Train Framework（自研）
模型规模	>10B至>100B
输出格式	原始权重（FP32/FP16）→ 量化（INT8/INT4）→ 打包为.cai
训练周期	根据模型规模，数周至数月
输出模型	基础模型（Foundation Model），供下游微调
3.2 分布式训练（CNBP集群）
对于中等规模模型（1B-10B），利用CLSDN组网的多台CNBP进行分布式训练。

属性	说明
硬件底座	多台CNBP通过CLSDN组网
训练框架	CAI Distributed Trainer
模型规模	1B-10B
数据并行	支持（数据分片到各节点）
模型并行	支持（大模型拆分到多节点）
3.3 边缘微调（CGPU / CorePhone）
在工业现场或移动端，利用本地数据进行模型微调。

属性	说明
硬件底座	CGPU、CorePhone、CoreBook
训练框架	CAI Edge Tuner（轻量版）
微调规模	1B以下模型
数据隐私	数据不离设备
更新方式	微调后生成增量包，上传至Core Cloud

五、分发层
5.1 Core Cloud 模型仓库
所有正式发布的CAI模型存储在Core Cloud模型仓库中

设备可通过CoreID认证后下载所需模型

模型下载自动验证数字签名

5.2 CNS 算力网络分发
对于需要实时推理的场景，CNS可直接将模型下发至执行设备

边缘设备（CGPU）可向CNS请求模型更新

支持差分模型分发（仅传输变化部分）

5.3 OTA 增量更新
模型更新以增量包形式分发（不整模型替换）

支持静默更新（后台下载，下次重启生效）

更新失败自动回滚到上一版本

六、安全层
6.1 模型签名与验证
每个.cai模型文件包含数字签名

签名由CRADT签署（研发团队），CPT确认（归档团队）

模型加载时自动验证签名，无效签名拒绝加载

验证失败触发安全事件并记录至EEPRAM（如果设备配备）

6.2 推理审计与追溯
每次推理记录：设备ID + 时间戳 + 模型版本 + 输入摘要

审计日志存储于EEPRAM（符合DOC-2026-027规范）

工业场景下，推理决策可追溯至具体模型版本和输入数据

6.3 访问控制与权限管理
模型权限与CoreID绑定

不同设备有不同的模型加载权限

敏感模型（如工业安全相关）需要额外授权

权限校验在模型加载时自动执行

8.2 版本管理策略
版本类型	说明	兼容性
主版本（X.0.0）	重大架构变更	不向后兼容
次版本（x.Y.0）	功能改进	向后兼容（模型兼容）
补丁版本（x.y.Z）	错误修复、性能优化	完全兼容
8.3 社区贡献机制
社区开发者可训练模型并提交至CRADT审核

审核通过后纳入Core Cloud模型仓库

社区模型在CAI Studio中标记为“社区贡献”

贡献者获取共建者股份（详见DOC-2026-002）
