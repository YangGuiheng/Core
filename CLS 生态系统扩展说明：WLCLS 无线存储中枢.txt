一、概述
1.1 定义
本文档是对 DOC-2026-011-CLS《CoreLink Server 技术规范》的生态系统扩展说明，定义CLS体系中存储侧的无线扩展方案，包括WLCLS（无线链路中枢）和WLHBM（无线存储模块）。

1.2 文档定位
属性	说明
文档类型	生态系统扩展说明
关联主文档	DOC-2026-011-CLS（CoreLink Server 技术规范）
核心内容	CLS与WLCLS/WLHBM的协同工作方式
目的	阐明CLS从“节点间互联”到“节点+存储”的完整拓扑
1.3 解决的问题
CLS原有文档定义了多台CNBP之间的高速互联，但未定义单台CNBP如何无线扩展存储。本扩展说明填补了这一空白，并阐明WLCLS+WLHBM如何以无线方式接入CLS体系。

三、CLS 与 WLCLS 的协同工作
3.1 光信号兼容性
WLCLS的上行光信号与CLS的激光通信使用相同的物理层标准：

相同波长窗口（绿光对准 + 红光/不可见光数据传输）

相同协议栈（CoreLink协议）

相同速率（100GB/s）

这意味着：WLCLS可以被CLS集群识别为一个“存储节点”，而不是一个独立的外设。

3.2 集群中的角色
节点类型	角色	数量
CNBP（主控）	CLS集群主节点	1台
CNBP（工作节点）	CLS集群计算节点	4台
WLCLS	CLS集群存储中枢	1台（可多台）
WLHBM	WLCLS管理的存储节点	最多8个/台WLCLS

四、典型部署场景
4.1 场景一：集群共享存储池
配置：1台主控CNBP + 4台工作CNBP + 1台WLCLS + 8个WLHBM

连接方式：

CLS激光互联所有CNBP节点

主控CNBP通过光信号连接WLCLS

WLCLS通过毫米波管理8个WLHBM

效果：所有CNBP节点通过CLS访问主控，再通过WLCLS访问存储池

4.2 场景二：单机无线存储扩展
配置：1台CNBP + 1台WLCLS + 1个WLHBM

连接方式：

CNBP通过毫米波（50GB/s）连接WLCLS

WLCLS管理单个WLHBM

效果：CNBP获得不占用有线接口的高速无线存储扩展

4.3 场景三：数据摆渡站
配置：1台WLCLS（内置256GB HBM缓存）+ 多个WLHBM

用途：

作为物理隔离环境的数据中转站

WLCLS通过光信号/毫米波接收数据，暂存于HBM缓存

再分发至多个WLHBM节点

五、与CLS的协议兼容性
层级	CLS原生	WLCLS扩展
物理层	激光（红/绿光）	光信号（兼容）+ 毫米波（扩展）
数据链路层	CoreLink协议	相同协议，增加存储管理命令
网络层	1+4星型拓扑	增加存储节点识别和路由
应用层	节点间高速互联	增加存储池管理和读写命令

七、结语
CLS不仅定义了节点间的高速互联，在本扩展文档的补充下，它完整定义了节点间互联 + 节点与存储互联的双层无线架构：

CLS：节点与节点之间

WLCLS：节点与存储池之间

WLHBM：存储池中的单个存储节点

三者通过光信号和毫米波连接，形成了完整的三层无线存储与通信体系。
