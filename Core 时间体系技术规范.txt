一、概述

1.1 定义

Core 时间体系是 Core 生态内跨天体、跨设施的统一时间基准与本地时间换算规范。它由 Galactic Standard Time（GST）作为统一基准，各天体或设施基于自身自转周期定义本地时间，并在显示层进行换算。

1.2 设计原则

原则	说明
统一基准	所有时间戳以 GST 秒数存储，避免跨天体歧义
本地呈现	用户端按所在天体的本地习惯显示时间
物理一致	GST 与 SI 秒一致，无需额外转换
可换算	任意天体本地时间可转换为 GST 秒数，再转换为任意其他天体本地时间
二、Galactic Standard Time（GST）

2.1 定义

GST 是 Core 生态的统一时间基准，以 SI 秒为单位，与任何天体的自转周期解耦。

定义：

1 GST 秒 = 1 SI 秒（国际单位制秒）
所有 Core 系统的时间戳以 GST 秒数存储
GST 不受任何天体自转、公转或历法影响
2.2 GST 的用途

Core 系统日志时间戳
CSN 通信协议的时间同步
Core Bank 交易记录
跨天体事件协调
所有需要跨天体一致性的时间记录
三、本地时间

3.1 定义

本地时间是某一特定天体或设施按照自身自转周期定义的时间显示方式。每个天体独立定义其本地时间，不与其他天体同步。

3.2 本地日的定义

天体/设施	自转周期	1 本地日
地球	24h 00m 00s	24 小时
火星	24h 37m 22s	24h 37m 22s
木星	9h 55m 30s	9h 55m 30s
土星	10h 33m 38s	10h 33m 38s
月球	27.32 天	27.32 地球日
空间站（CSS/CIN）	无自然日	按 GST 显示或自定轮班周期
3.3 本地时间的 0 点

每个天体的本地时间 0 点由该天体的子夜时刻定义。具体实现方式由部署在该天体的 Core 设施根据当地天文条件确定。

四、时间换算

4.1 换算规则

任何时间点都可以表示为 GST 秒数。本地时间与 GST 之间的换算是通过目标天体的自转周期比例进行的。

换算步骤：

目标天体本地时间 → 转换为 UTC（或本地时间） → 转换为 GST 秒数
GST 秒数 → 计算目标天体当前在自转周期中的位置 → 显示为目标天体本地时间
从火星本地时间换算为木星本地时间时，先转为 GST 秒数，再按木星自转周期计算
换算公式（概念层）：

text
本地时间秒数 = GST 秒数 % 本地日秒数
4.2 示例

火星本地 13:00 → 转换为木星本地时间

火星本地 13:00 对应一个特定的 GST 秒数
将 GST 秒数按木星自转周期（9h55m30s）计算
显示为木星本地时间
五、显示规则

5.1 默认显示

使用场景	默认显示
用户登录所在天体	该天体本地时间
跨天体通信	发送方本地时间 + 接收方本地时间（可切换）
系统日志	GST 秒数（或可读 GST 时间）
Core Bank 交易记录	GST 秒数
用户日常界面	当前所在天体本地时间
5.2 显示切换

用户可在支持 GST 的天体本地时间之间自由切换，系统按当前所在天体自动调整，也可手动设置为任意支持天体的本地时间。

5.3 时间格式

年月日时：使用该天体本地习惯（如火星本地日历可与地球同步，或以火星年为单位）
时区字段：显示为“GST”或天体代号（如“MST”表示火星本地时间）
标识	含义
GST	Galactic Standard Time
EST	Earth Standard Time（地球本地时间）
MST	Mars Standard Time（火星本地时间）
JST	Jupiter Standard Time（木星本地时间）
SST	Saturn Standard Time（土星本地时间）
