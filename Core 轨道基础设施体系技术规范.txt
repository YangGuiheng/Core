一、概述

1.1 定义

Core 轨道基础设施体系是 Core 在太阳系尺度上部署的物理资产运维架构，由以下层级组成：

层级	实体	职责
管理部门	Core Aviation Department（CAD）	轨道资产日常运营管理
执行团队	Core Joint Maintenance Team（CJMT）	节点维护与维修执行
维护平台	Core Space Station（CSS）	轨道维护基地
物流枢纽	Core Integrated Node（CIN）	燃料补给与物资中转
1.2 设计原则

原则	说明
分级维护	一般节点与关键节点差异化配置维护团队
就地取能	CSS 利用当地能源（光能/热感电），CIN 自备核聚变
休眠值守	远空 CSS 大部分时间无人值守，冬眠乘组按需唤醒
物资闭环	备件与故障节点定期轮换，不积累废弃物资
分级物流	CSS 服务于节点维护，CIN 服务于跨区域物资转运
二、Core Aviation Department（CAD）

2.1 定位

CAD 是 Core 轨道资产的日常运营管理机构，直接向 Core 基金会汇报。

2.2 核心职责

职能	说明
星座运维	监测 CSN 全部节点健康状态，执行日常运维指令
轨道调节	执行轨道保持、变轨、避让等机动操作
碰撞规避	监测空间碎片，协调与其他航天器安全距离
频谱管理	管理 Core 使用的无线电频率资源
Core Rocket 发射执行	发射窗口计算、飞行控制、入轨部署、测控跟踪
CJMT 调度	维护团队的任务规划、乘组轮换与调度
2.3 协作关系

协作方	内容
Core Industrial Sector（CIS）	Core Rocket 建造与硬件制造
Core Star Net（CSN）	通信协议与数据中继
Core Joint Maintenance Team（CJMT）	维护任务执行
三、Core Joint Maintenance Team（CJMT）

3.1 定位

CJMT 是 CAD 与 CIS 联合组建的维护执行团队，由 CAD 统一管理。

3.2 团队配置

一般节点维护团队：

角色	人数	说明
远程遥控师	2	地面远程操作，非触控式维护/诊断/软件更新
轨道器操作员	2	轨道器平台操作控制（机动、对接、能源管理）
外层空间宇航员	2	舱外作业，直接接触节点硬件
合计	6	—
关键节点维护团队：

角色	人数	说明
远程遥控师	3	地面远程操作
轨道器操作员	3	轨道器平台操作控制
航天器机械臂操作员	3	机械臂辅助作业
Core Space Station 驻轨工程师	2	在轨技术协调
外层空间专业维护人员	3	复杂舱外作业
合计	14	—
3.3 分级标准

节点类型	判断依据	维护团队配置
一般节点	中继节点、采集节点、普通通信节点	6人团队
关键节点	核心骨干节点、区域汇聚节点、深空中继、L1聚变电站	14人团队
四、Core Space Station（CSS）

4.1 定位

CSS 是 CAD 的轨道维护平台，分布在各关键节点附近，作为 CJMT 在轨驻留与作业基地。

4.2 分布

区域	覆盖范围
近空	地球轨道、月球轨道、火星轨道
远空	木星、土星、天王星、海王星、冥王星
4.3 标准舱段配置

CSS 由以下舱段组成：

舱段	功能
推进舱	轨道维持与机动
服务舱	平台基础服务（供电、热控、通信）
轨道机动舱	独立轨道平台，用于 CSS 无法到达区域的捕获-维修-释放作业
生态舱	空气循环、水循环、食物产出
休眠舱	乘组长期冬眠（远空 CSS 核心舱段）
控制舱	站务管理与指挥
储物舱	物资存储
对接舱	CR 对接
生活舱	乘组日常生活（交接期使用）
工作舱	维护作业与操作
医疗舱	乘组健康保障与应急医疗
EVA 舱（气闸）	舱外活动支持
运动舱	乘组体能训练
备件舱	专用备件存储
辐射防护区	太阳风暴期间乘组紧急庇护
4.4 技术规格

属性	规格
休眠人员	40 人
最大常驻维持	100 人
CR 对接口	2 个
重力	无
近空能源	光能（太阳能）
远空能源	热感电（利用行星热辐射/温差发电）
4.5 运营模式

近空 CSS（地球-火星）：

阶段	状态	持续时长
交接期	有人驻轨	约 1 年
值守期	乘组驻轨值班	约 1 年
总驻轨周期	—	2 年
远空 CSS（木星-冥王星）：

阶段	状态	持续时长
交接期	有人驻轨（新乘组培训与交接）	约 1 年
值守期	冬眠（休眠）	约 9 年
总驻轨周期	—	10 年
远空 CSS 唤醒机制：

CSS 自主监测系统持续运行
仅在发生超出系统自主决策能力的事件时唤醒人员
逐层唤醒：技术员 → 工程师 → 指挥官
问题解决后，如无持续任务，乘组恢复冬眠
4.6 乘组交接与物资流转

新乘组携带：

携带物资	说明
燃料	供 CSS 轨道维持
生态物质	水、土壤、种子
备用 CSN 节点	5 颗
最新备件	替换 CSS 旧备件
原乘组返回携带：

携带物资	说明
旧备件	确保 CSS 内存放最新备件
故障 CSN 节点	已替换的有问题节点
未使用备用节点	如有剩余，一并带回
原则：CSS 内存放的始终是“最新”备件，不积累过时物资。

五、Core Integrated Node（CIN）

5.1 定位

CIN 是 Core 在太阳系中的大型综合物流枢纽，是轨道交通网络的补给与中转核心。共部署 4 个，分布在燃料经济性最优的轨道位置。

5.2 分布方向

编号	位置	说明
CIN-01	内太阳系	地球-火星之间
CIN-02	小行星带	小行星带附近
CIN-03	外太阳系	木星轨道附近
CIN-04	深空	土星或天王星轨道附近
5.3 技术规格

属性	规格
总容量	500 人（含流动人口）
常驻人员	50 人
休眠人员	50 人
CR 对接口	10 个
重力	有（通过离心力实现）
能源	核聚变
轨道维护能力	优于 CSS
科学实验能力	大规模轨道科学实验
生态系统	比 CSS 更完整
5.4 与 CSS 的对比

维度	CSS	CIN
定位	节点维护站	交通枢纽/补给站
分布密度	多站点（分布在关键节点附近）	4 个（分布在关键轨道位置）
最大容量	100 人	500 人
对接口	2 个	10 个
重力	无	有（离心力）
能源	光能/热感电	核聚变
主要功能	CSN 节点维护	CR 补给、物资中转、物流协调
5.5 CIN 的物流职能

为途经 CR 补充燃料，使其有足够燃料到达目标 CSS
接收从远空 CSS 返回的 CR，进行补给
接收返回物资（旧备件、故障节点），进行初步分析后发回地球
储存新备件，供后续 CR 携往 CSS
协调跨区域物资分配
