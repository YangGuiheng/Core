一、概述

1.1 定义

Core 地方治理体系（CGC，Core Governance Committee）是 Core 生态内自下而上的地方治理结构，由 Grassroots、Block、Region、Sar、Unite 五级委员会组成。各级 CGC 由下级选举产生，对下级负责，负责处理各自辖区内的日常事务与区域协调事务。

1.2 设计原则

原则	说明
自下而上	权力来源于基层，上级由下级选举产生，对下级负责
按比例代表	代表数量按人口比例产生，确保代表性
分级决策	不同层级处理不同规模的事务
人事独立	罢免与任免由产生方决定
紧急适应	紧急状态下可调整决策门槛
二、五级结构

2.1 层级划分

层级	名称	说明
第1级	Grassroots（基层）	最小的治理单元，由辖区内的 CID 持有者直接选举产生
第2级	Block	由多个 Grassroots 组成，由下级 Grassroots 选举产生
第3级	Region	由多个 Block 组成，由下级 Block 选举产生
第4级	Sar	由多个 Region 组成，由下级 Region 选举产生
第5级	Unite	该星的最高 CGC 机构，由下级 Sar 选举产生
2.2 代表比例

各级 CGC 的代表数量按以下比例产生：

层级	产生方式	比例
Grassroots	由辖区 CID 持有者直接选举	辖区人口的 25%
Block	由下级 Grassroots 选举	Grassroots 代表人数的 25%
Region	由下级 Block 选举	Block 代表人数的 25%
Sar	由下级 Region 选举	Region 代表人数的 25%
Unite	由下级 Sar 选举	Sar 代表人数的 25%
说明：以地球约 70 亿人口为例，经过五级 25% 的递进筛选，Unite CGC 的人数依然是一个足够形成有效委员会的规模。比例递进机制不导致高层代表人数不足。

三、选举资格

3.1 基本资格

资格	投票	参选
持有有效 CID	✅ 必须	✅ 必须
社会权限 ≥ 6	✅ 必须	✅ 必须
年满 18 周岁	✅ 必须	✅ 必须
选举资格由社会权限决定，不因居住地、职业或家庭身份而改变。个人权限和家庭权限不参与选举资格判断。

3.2 选举周期

层级	选举周期	说明
Grassroots	每年	基层代表
Block	每两年	由下级 Grassroots 选举产生
Region	每两年	由下级 Block 选举产生
Sar	每三年	由下级 Region 选举产生
Unite	每三年	由下级 Sar 选举产生
不同层级的选举周期错开，形成“基层高频、高层低频”的节奏，确保基层事务能够被及时响应，同时保持高层决策的连续性。

四、事务权限码在 CGC 事务中的应用

各级 CGC 事务中的权限分配遵循 DOC-2026-047-ADD1 定义的事务码机制：

在 CGC 中的角色	事务码	说明
CGC 代表（参与投票决策）	6	可读+可写（了解情况并参与表决）
CGC 列席者（观察员）	4	可读（了解情况，不参与表决）
CGC 执行者	5	可读+可执行（了解决议并执行）
CGC 记录员	2	可写（记录会议内容，不参与决策）
CGC 旁听者	1	可执行（执行旁听程序，无其他权限）
事务码在具体事务中被授予，随事务不同而变化，事务结束后自动失效。

五、决策机制

5.1 决策门槛

决议类型	通过门槛	适用场景
基础决议	≥ 1/2	日常事务、常规决策
重要决议	≥ 2/3	预算、政策调整、人事任免
重大决议	≥ 3/4	结构性变更、章程修订
紧急状态决议	≥ 1/2	紧急状况下的快速响应
5.2 紧急状态的定义

紧急状态指 CGC 代表存在以下行为之一：

操控选举
滥用职权
对产出单位或其他单位的恶意行为
其他严重影响 CGC 正常运行的行为
紧急状态下，被指认的代表可被 1/2 门槛通过罢免，以便快速处理严重问题。

5.3 决策权限范围

层级	决策范围
Grassroots	社区日常事务、邻里纠纷、基层资源分配
Block	多个 Grassroots 之间的协调事务
Region	跨 Block 事务、区域发展规划
Sar	跨 Region 事务、区域重大决策
Unite	跨 Sar 事务、行星级重大决策
六、人事任免与问责

6.1 任免原则

各级 CGC 代表的任免由产生其的单位决定：

层级	任免主体
Grassroots	由辖区 CID 持有者直接决定
Block	由下级 Grassroots 决定
Region	由下级 Block 决定
Sar	由下级 Region 决定
Unite	由下级 Sar 决定
6.2 罢免门槛

常规罢免：需达到该级 CGC 全体代表的 2/3 通过
紧急罢免：需达到该级 CGC 全体代表的 1/2 通过
6.3 问责链条

各级 CGC 向下负责，形成层层问责的链条：

text
Grassroots → 对辖区 CID 持有者负责
Block → 对产出它的 Grassroots 负责
Region → 对产出它的 Block 负责
Sar → 对产出它的 Region 负责
Unite → 对产出它的 Sar 负责
6.4 Core Foundation 的兜底介入

当某一级 CGC 存在严重不作为，且其产生单位未能及时采取措施时，Core Foundation 有权进行适度干预。

介入条件：

该级 CGC 存在明确的不作为或违规行为
其产生单位未在合理期限内启动罢免或纠正程序
该行为已影响到 Core 核心原则或广泛群众利益
介入方式：

向该级 CGC 及其产生单位发出正式警告
限期要求其产生单位启动罢免或纠正程序
若限期未响应，Core Foundation 可直接介入处理
Core Foundation 的介入以“恢复 CGC 正常运行”为限度，不介入 CGC 的日常自治事务。

七、CGC 与 Core Foundation 的关系

事务类型	处理主体
日常地方事务	CGC 自行处理
跨区域协调事务	上级 CGC 协调处理
涉及 Core 核心原则的事务	CGC 决策 + Core Foundation 确认
文明级事务	Core Foundation + 熔断机制
CGC 严重不作为	Core Foundation 兜底介入
CGC 的决策在各自权限范围内独立，不受 Core Foundation 直接干预，但 CGC 的决策不得违背 Core 的核心原则（如宣言、转型原则、熔断与授权机制）。若 CGC 的决策违反 Core 核心原则，Core Foundation 有权提出异议，并要求 CGC 重新审议。
