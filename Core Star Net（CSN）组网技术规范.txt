一、概述
1.1 定义
Core Star Net（CSN）是 Core 在太阳系尺度上的通信与组网基础设施。它将 Core 的地面网络（CLSDN）延伸至整个太阳系，实现从太阳到柯伊伯带的全面覆盖。

1.2 与 Core 现有网络的关系
网络	覆盖范围	定位
CoreLink	单设备间	短距高速互联
CLSDN	地面/近地	多设备组网与存储网络
CSN	太阳系尺度	星际通信骨干
CSN 是 CLSDN 的星际延伸。对上层应用而言，通过 CSN 通信与通过 CLSDN 通信没有区别——数据包的路由、寻址、转发由网络层统一处理，底层是地面链路还是星际链路对用户透明。

1.3 设计目标
目标	说明
太阳系全域覆盖	从太阳附近到柯伊伯带，所有区域均可接入
自主路由	节点自主决策数据路径，不依赖地面控制中心
容迟网络	支持存储转发，适应数小时级延迟
渐进部署	从内太阳系向外扩展，每一层独立运行
与CLSDN兼容	使用相同的寻址和协议体系，无缝切换

2.2 层级扩展规则
每向外一层，节点数量扩展至上一层的约3倍：

层级	区域	节点数量（约）	扩展倍数
0	太阳高轨	8	—
1	水星-金星轨道	~960	120×
2	地球-火星轨道	~2,800	3×
3	小行星带	~8,400	3×
4	木星轨道	~25,000	3×
5	土星轨道	~75,000	3×
6	天王星轨道	~225,000	3×
7	海王星轨道	~675,000	3×
8	柯伊伯带	~2,000,000	3×
注：第1层从8扩展到960为初始部署的集中投入阶段，后续层级的3倍扩展是稳定的递进模式。

2.3 节点类型
类型	功能	部署位置
广播网关	太阳系级广播，协议同步	太阳高轨
骨干节点	高速数据中继，跨区域路由	行星轨道
区域汇聚节点	本地数据汇聚、存储、处理	行星轨道
边缘节点	终端接入、数据缓存	各轨道层
信标节点	导航、测距、时间同步	全区域
三、通信链路
3.1 链路类型
链路类型	介质	速率	距离	用途
星间激光链路	激光	100-400 Gbps	≤5,000 km	相邻节点间高速骨干
星际激光链路	激光	10-100 Gbps	0.5-5 AU	层间中继、跨区域通信
引力波/透镜广播	引力/射电	低速率	无限	太阳系级广播
微波链路	射频	1-10 Gbps	自适应	终端接入、备用链路
激光终端直连	激光	1-10 Gbps	≤1 AU	Core设备直连卫星
3.2 链路预算
通信段	典型距离	延迟	速率（典型）
同一轨道层相邻节点	100-1,000 km	0.3-3 ms	100-400 Gbps
相邻轨道层之间	0.5-2 AU	4-16 min	10-100 Gbps
内太阳系↔外太阳系	2-5 AU	16-40 min	10-50 Gbps
太阳↔柯伊伯带	30-50 AU	4-6.5 h	1-10 Gbps
太阳引力透镜广播	550 AU（焦点）	极长	低速率
3.3 路由协议
CSN 采用分段路由 + 存储转发的混合机制：

区域	路由方式	说明
同一轨道层内	实时路由	节点间距小，延迟可忽略，使用类似地面网络的动态路由
相邻轨道层之间	调度路由	利用轨道运动的可预测性，预计算传输窗口
跨区域（内↔外）	存储转发	距离远、延迟大，数据包沿节点链逐跳存储转发
太阳系级广播	广播路由	通过引力透镜广播网关一次性覆盖全区域
3.4 容迟网络（DTN）机制
CSN 针对星际通信延迟实现完整的容迟网络架构：

存储转发：每个节点具备足够的存储能力，可在链路中断时缓存数据，等待下一次通信窗口。

托管传输：数据包携带端到端元数据，中间节点自主决定转发时机和路径，无需端到端确认。

逐跳确认：每段链路独立确认，而非端到端确认。发送节点仅需确认下一跳已接收，即可释放本地缓存。

链路状态预测：节点根据轨道力学和星历数据，预计算未来一段时间内可用的通信窗口和链路质量。

四、寻址与命名
4.1 节点地址格式
CSN 节点地址采用分层结构：

text
[区域].[轨道层].[节点ID].[域]
示例：

SOL.L2.1047.SRV — 太阳系，第2轨道层（火星区域），节点1047，服务节点

SOL.L4.0823.GW — 太阳系，第4轨道层（木星区域），节点823，网关节点

SOL.L0.0001.BC — 太阳系，第0轨道层（太阳高轨），节点0001，广播网关

SOL.L2	地球-火星轨道	第2层

五、与 Core 生态的集成
5.1 与 CLSDN 的关系
text
Core 设备（CNBP、CGPU、CorePhone）
   │
   ├── 有地面网络 → CLSDN（地面组网）
   │
   ├── 无地面网络 → CSN 终端直连（激光/微波）
   │                      │
   │                      ▼
   │               CSN 边缘节点
   │                      │
   │                      ▼
   │               CSN 星际骨干
   │
   └── 任意位置 → 自动选择可用网络（对用户透明）
5.2 与 CNS 的关系
CSN 节点可承载 CNS 的算力调度功能：

功能	说明
算力注册	太阳系内任意 CNBP 可通过 CSN 向 CNS 注册
任务调度	CNS 可通过 CSN 将任务调度到任意位置的 CNBP
状态同步	CNS 的算力池状态通过 CSN 在太阳系内同步
5.3 与 Core Cloud 的关系
CSN 节点搭载 Core Cloud 分布式存储副本

太阳系内任意 Core 设备可通过 CSN 存取数据

关键数据在多个轨道层保留冗余副本

5.4 与 CAI 的关系
CSN 节点搭载 CAI 1.5B（深空优化版），支持在轨数据处理

数据在产生位置附近完成初步处理，减少下行带宽占用

模型通过 CSN 进行太阳系级同步更新

六、节点功能分层
6.1 各层节点功能

L0（太阳高轨）	系内广播、时间基准、协议同步	太阳活动监测
L1（水星-金星）	内太阳系高速骨干、近太阳数据中继	科学数据采集
L2（地球-火星）	Core 核心区通信、地面互联	深空任务支持
L3（小行星带）	内外太阳系中继、资源数据采集	区域路由
L4（木星）	外太阳系核心节点、区域计算中心	跨区域中继
L5（土星）	外太阳系辅助节点	科学数据存储
L6（天王星）	深空中继节点	边缘数据缓存
L7（海王星）	深空边缘节点、数据存储	星际边界监控
L8（柯伊伯带）	星际信标、边界网关	深空导航
6.2 核心节点功能定义
节点角色	主要职责	部署位置
广播网关（BC）	太阳系级广播发送与接收、协议同步、时间基准分发	L0（太阳高轨）
核心骨干（BB）	跨区域高速数据传输、路由决策、网络状态同步	L1-L8各层骨干节点
区域中心（RC）	本地数据汇聚、存储、处理、服务本地Core设备	各行星轨道
接入节点（AP）	终端设备接入（Core设备直连）、本地缓存	各层边缘
信标节点（BK）	导航定位、测距、时间同步	全区域
