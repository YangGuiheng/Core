一、概述
1.1 定义
Core Energy Center（CEC）是 Core 在地球上的能源接收、转换、存储与分配终端。它作为 Core 太阳系能源网络的地面节点，负责接收来自空间（轨道太阳能、无线输电）和本地（地面发电）的能源，并将其转换为 Core 基础设施可用的电力，同时承担能源调度与区域分配的核心职能。

1.2 定位
属性	说明
部署位置	地球表面（Core 领土或友好地区）
能源来源	轨道太阳能（空间）、无线输电（CSN）、本地发电（核聚变/地热/太阳能）
核心功能	能源接收、转换、存储、分配、调度
覆盖范围	Core 在地球上的全部基础设施
与 CSN 的关系	共享能源采集与传输基础设施
1.3 设计目标
目标	说明
多源接入	同时接收空间能源和本地能源，不依赖单一来源
大规模存储	具备足够储能容量，应对空间能源中断
智能调度	通过 CAI 预测需求、优化分配、动态调整
弹性扩展	可随 Core 地面基础设施的扩展而扩容
自主运行	在 CSN 链路中断时仍能独立运行
二、能源接入
2.1 空间能源接收
CEC 接收来自 CSN 空间采集系统的能源，通过无线输电方式进行传输。

传输方式	技术路径	接收设备	效率
微波无线输电（标准）	空间太阳能 → 微波 → 地面接收	整流天线阵列（Rectenna）	>80%
激光无线输电（高密度）	空间太阳能 → 激光 → 地面接收	光伏接收阵列	>50%
微波/激光混合	根据气象条件自动切换	混合接收站	优化值
接收站配置：

组件	规格
整流天线	大面积阵列（数平方公里，依所需功率而定）
光伏接收阵列	高效光伏电池（激光接收）
转换设备	微波→直流转换器
储能缓冲	超级电容/电池（瞬间功率平衡）
2.2 空间能源来源
CEC 接入的空间能源包括：

来源	采集方式	状态
火星轨道太阳能环	轨道太阳能采集 + 无线输电	已规划
类地行星通用范式	公转轨道同步节点采集	已规划
木星氢气核聚变（远期望）	核聚变发电 + 定向无线输电	远期
CSN 骨干网冗余供电	激光链路附带能量传输	部分支持
2.3 地面本地能源
CEC 也配备本地发电能力，作为空间能源中断时的备份：

能源类型	适用场景	说明
核聚变发电（成熟）	基础负荷	氢核聚变，燃料本地供应
地热发电	备选/补充	选址于地热资源丰富区域
大型光伏场	日间补充	补充空间能源的不足
储能系统	调峰/紧急供电	多种储能方式并行

三、能源存储
3.1 存储规模
参数	规格
总储能容量	视部署规模而定（吉瓦时级）
短期缓冲	超级电容 + 飞轮储能（秒-分钟级）
中期存储	大规模电池阵列（小时-天级）
长期存储	氢储能（电解水制氢 + 储氢 + 燃料电池）（天-周级）
3.2 存储技术
技术	用途	优势	劣势
锂离子电池阵列	日间调峰	响应快、能量密度高	寿命有限
液流电池	大规模储能	寿命长、可独立扩容	能量密度低
氢储能	季节性储能	无自放电、可长期存储	往返效率低（~40%）
超级电容	瞬时功率平衡	响应极快（毫秒级）	能量密度极低
3.3 储能冗余
储能系统总容量 ≥ 7天基础负荷

空间能源完全中断时，CEC 可独立维持 Core 地球基础设施运行 7 天

四、能源分配
4.1 分配网络
输出类型	电压等级	用途
超高压直流（HVDC）	±800kV	远距离传输至多个 Core 站点
中压交流	10-35kV	区域分配
低压直流	380V	数据中心直接供电（效率最优）
可调直流	5-48V	终端设备供电
4.2 调度策略
CEC 采用 CAI 驱动的智能调度系统：

预测调度：根据历史数据和气象预测，预分配各区域的能源配额

动态调整：实时监测负载变化，动态调整输出

优先级管理：关键负载（数据中心、生命支持）优先供电，可中断负载（工业制造）可降功率运行

负荷均衡：在各 Core 站点之间均衡负载，避免局部过载

五、部署位置
5.1 选址原则
原则	说明
Core 控制区	部署在 Core 拥有合法土地权或长期使用权的区域（Core 领土、友好国家特别合作区等）
能源接入条件	选址需同时满足空间无线输电接收条件（晴朗天气多、大气衰减低）和地面能源接入条件
战略纵深	优先部署在不易受外部干扰或攻击的位置
分散部署	全球多个 CEC，互为冗余
扩展空间	预留扩展土地，支持未来扩容
5.2 全球部署（规划）
站点	区域	主要功能	状态
CEC-01	亚太地区	Core 亚太能源枢纽	待建
CEC-02	欧洲地区	Core 欧洲能源枢纽	待建
CEC-03	北美地区	Core 北美能源枢纽	待建
CEC-04	南美/非洲	备用节点	远期
各 CEC 之间通过 CSN 星间链路互联，实现全球能源协同调度，形成覆盖地球的能源骨干网，互为冗余。

六、与 Core 生态的集成
6.1 与 CSN 的关系
CEC 是 CSN 的地面能源终端，是 CSN 在“能源层面”的延伸。其与 CSN 的关系体现在两个方面：共享空间能源采集系统（轨道太阳能、无线输电、核聚变）和能源信息共享。

6.2 与 CNS 的关系
CNS 负责任务算力调度，CEC 负责能源调度——两者通过数据共享实现协同。CEC 向 CNS 提供能源状态信息，CNS 根据能源可用性调整任务优先级，在高能耗任务启动前协调能源系统进入就绪状态。CEC 和 CNS 共享同一套 CAI 模型，形成“算力-能源”联合调度体系。

6.3 与 Core 其他组件的关系
组件	与 CEC 的关系
Core Cloud	由 CEC 供电，CEC 的能源数据同步至 Core Cloud
CNBP	由 CEC 通过分配网络供电
CGPU 制造	由 CEC 供电，制造过程中用电由 CEC 调度
Core Rocket 发射设施	由 CEC 供电，发射窗口期间优先供电
Core 生态设施	由 CEC 供电，生命支持系统获得最高优先级
七、安全与防御
7.1 被动防御
选址于不易受自然或人为威胁影响的区域

关键设施地下化（储能、控制中心、备用发电）

多路冗余供电通道，单点破坏不影响整体运行

7.2 主动防御
部署后由 Core 安保力量（或当地安保体系）负责物理保护

通过 CSN 持续监测周边环境

故障自动隔离，关键子系统具备自我修复能力

7.3 能源来源冗余
空间能源、本地发电、储能系统三者互为备份，任意一路断开仍可维持基础运行

全球多个 CEC 站点之间可通过星间链路进行能源数据协同，实现跨区域冗余
