一、概述
1.1 定义
Core Star 是 Core 在太空中的标准化卫星平台，是构成 Core Star Net 的基本物理单元。它采用通用平台设计，具备标准的能源、计算、通信和载荷接口，可根据任务需求挂载不同载荷，部署于太阳系各轨道层。

1.2 核心定位
属性	说明
产品名称	Core Star
类型	标准化卫星平台
部署区域	太阳系全域（从太阳高轨到柯伊伯带）
核心计算	CGPU Star特别版（搭载CAI 1.5B）
能源系统	4×太阳能板
载荷接口	顶部挂载点（标准化接口）
任务角色	CSN节点、通信中继、数据处理、科学探测、深空信标
1.3 设计原则
原则	说明
通用平台	单一硬件平台适配多种任务，载荷可更换
标准化接口	所有子系统采用标准接口，可独立升级
在轨智能	内置CGPU Star，具备在轨数据处理和自主决策能力
能源自给	太阳能供电，根据部署区域适配不同能源方案
渐进部署	同一平台适用于内太阳系和外太阳系，仅载荷和能源模块差异
二、硬件规格
2.1 平台结构
参数	规格
平台类型	标准化卫星总线
干质量（不含燃料）	约200-300kg（取决于载荷配置）
发射质量（含燃料）	约300-500kg
设计寿命	10-15年（内太阳系）/ 20年+（外太阳系，核能版本）
姿态控制	3轴稳定 + 反作用轮 + 星敏感器
推进系统	电推进（离子/霍尔推进器）
2.2 能源系统
参数	规格
主能源	4×太阳能板
太阳能板布局	四周均匀分布（0°, 90°, 180°, 270°）
单板面积	约3-5m²（依任务需求调整）
总功率（内太阳系）	5-15kW
总功率（外太阳系）	1-5kW（核能补充）
储能	高容量蓄电池
外太阳系变体	搭载小型核反应堆（RTG/裂变堆）作为补充或主能源
2.3 计算核心：CGPU Star特别版
参数	规格
芯片型号	CGPU Star（太空加固版）
基础架构	基于CGPU（28nm成熟工艺，太空辐射加固封装）
CPU核心	4核 ARM Cortex-A55（太空级）@ 1.2GHz
NPU算力	1-2 TOPS（INT8）
内存	16GB DDR3（太空级，抗辐射）
存储	256GB eMMC（系统）+ 1TB SSD（数据）
预装AI模型	CAI 1.5B（太空优化版）
功耗	5-10W（典型）/ 15W（峰值）
抗辐射	支持单粒子翻转（SEU）检测与纠正
容错	双机冷备冗余（主备切换）
2.4 通信系统
参数	规格
星间链路	4×激光终端（2×同轨 + 2×异轨）
单链路速率	100-400 Gbps
对地链路	S频段 + X频段（备份）
深空链路	Ka频段（外太阳系）
终端直连	激光/微波终端（Core设备直连卫星）
2.5 载荷接口
参数	规格
位置	平台顶部
接口类型	标准化机械接口 + 电气接口（供电/数据）
挂载能力	≤100kg
供电输出	可配置（5V/12V/28V/48V）
数据传输	以太网/光纤（与CGPU Star直连）
载荷类型	通信载荷、遥感载荷、科学载荷、导航载荷等
三、任务载荷类型
3.1 标准载荷模块
Core Star 顶部挂载点可挂载以下标准载荷模块：

载荷类型	功能	典型应用
通信中继载荷	星间/星地高速通信	CSN骨干节点
光学遥感载荷	高分辨率成像	行星观测、资源勘探
科学探测载荷	磁场/粒子/辐射探测	深空科学研究
导航信标载荷	高精度授时与定位	深空导航参考点
引力波/透镜载荷	引力波探测、太阳透镜通信	太阳高轨广播网关
核能模块	小型核反应堆	外太阳系能源补充
3.2 载荷更换机制
载荷在发射前集成，发射后不可更换

同一平台设计支持不同载荷配置，可批量生产

载荷与平台之间的接口标准化，允许不同任务使用相同的平台批量制造，仅更换载荷模块即可适配不同任务需求

四、部署变体
Core Star 根据部署区域不同，有三种标准变体：

4.1 Core Star：内太阳系变体
参数	规格
部署区域	水星、金星、地球、火星轨道
主能源	4×太阳能板（标准面积）
补充能源	蓄电池
热控	被动热控 + 主动散热
典型任务	CSN节点、通信中继、地球/火星区域数据中心
4.2 Core Star Deep：外太阳系变体
参数	规格
部署区域	木星、土星轨道
主能源	4×太阳能板（增大面积） + 核能补充
补充能源	小型核反应堆（辅助）
热控	主动热控（加热器）
典型任务	外太阳系CSN节点、木星/土星区域数据中心
4.3 Core Star Edge：深空边界变体
参数	规格
部署区域	天王星、海王星、柯伊伯带
主能源	核能（裂变堆/RTG）
补充能源	太阳能（低功率，辅助）
热控	主动热控 + 同位素加热
典型任务	深空信标、边界网关、星际导航参考点
五、部署场景
5.1 Core Star Net 骨干节点
配置：通信中继载荷 + CGPU Star标准版

部署：各轨道层骨干位置

功能：星间高速数据中继、路由决策、网络状态同步

5.2 行星区域数据中心
配置：科学载荷 + 大容量存储 + CAI 1.5B

部署：行星轨道（如火星、木星）

功能：区域数据汇聚、在轨处理、本地存储

5.3 太阳引力透镜广播节点
配置：引力透镜载荷 + 大功率定向天线

部署：太阳高轨（L0层）

功能：太阳系级广播、深空导航、协议同步

5.4 深空边界信标
配置：导航信标载荷 + 核能模块 + 低功耗通信

部署：柯伊伯带

功能：深空导航参考点、星际边界标识、Core协议信标广播

六、与 Core 生态的关系
6.1 与 CSN 的关系
Core Star 是 CSN 的基本物理节点。每个 Core Star 部署后自动成为 CSN 的一部分，参与组网、路由和数据交换。

6.2 与 CNS 的关系
Core Star 可承载 CNS 的算力调度功能

外太阳系 Core Star 可作为 CNS 的远距离算力节点

Core Star 之间通过 CSN 共享算力池状态

6.3 与 CAI 的关系
每个 Core Star 搭载 CAI 1.5B（太空优化版）

支持在轨数据预处理和自主决策

可通过 CSN 接收模型更新

6.4 与 Core Rocket 的关系
Core Rocket 是 Core Star 的发射载具

每枚 Core Rocket 搭载一个或多个 Core Star

发射入轨后，Core Star 自主激活并加入 CSN
