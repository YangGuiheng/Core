一、概述

1.1 定义

Core RTOS Mobile 是 Core RTOS 的轻量化分支，专为低功耗移动设备（智能手机、智能手表、AR/XR眼镜、IoT设备等）设计。它在保留 Core RTOS 核心架构（RCRTOS + VCRTOS 双层设计）的同时，针对移动设备的功耗、存储和网络特性进行了深度优化。

1.2 与标准 Core RTOS 的差异

维度	Core RTOS（标准）	Core RTOS Mobile
目标设备	CNBP、工作站、服务器	CorePhone、CoreWatch、CoreGlass、IoT
功耗预算	≥100W	0.5W - 15W
存储占用	完整固件（~500MB）	精简固件（~50MB）
多OS支持	Windows、Linux、Android、CoreOS	Android、CoreOS Lite（轻量版）
更新策略	24h验证期 + 用户选择	低功耗自适应 + 夜间/闲置更新
网络依赖	持续在线（CLS/光信号）	间歇性连接（蜂窝/WiFi/BLE）
用户界面	Core Launcher（完整版）	Core Launcher Lite（精简版）
1.3 设计原则

原则	说明
极简内核	保留 Core RTOS 的核心安全与管理功能，去除桌面端特有模块
功耗优先	所有功能模块设计以低功耗为第一优先级
间歇性连接	适应移动设备不稳定的网络环境
本地决策	减少对 CNS 的持续依赖，关键决策在本地完成
快速启动	从上电到用户界面 < 100ms
二、架构精简

2.1 核心保留模块

模块	保留原因
RCRTOS + VCRTOS 双层架构	安全底座，不可妥协
CSMV 验证客户端	设备可信度评估
CAI 轻量版	本地行为分析（小型化模型）
Core Launcher Lite	设备启动器与状态面板
CLS 协议精简版	设备发现与轻量互联
2.2 移除/缩减模块

模块	处理方式	原因
多OS完整容器管理	移除（仅保留Android + CoreOS Lite）	移动设备不需要运行Windows/Linux
Windows/Linux 硬件直通	移除	同上
Core Launcher 完整版	替换为 Core Launcher Lite	降低内存占用与功耗
24h 验证期（标准版）	替换为低功耗自适应验证	移动设备无法保证持续在线24小时
文件级实时写入	保留，但仅在充电+WiFi时执行	移动设备需避免频繁存储操作
三、Core Launcher Lite

3.1 定义

Core Launcher Lite 是 Core RTOS Mobile 的用户界面，在操作系统启动之前显示，提供设备状态概览和启动控制。它在保留完整版 Core Launcher 核心功能的同时，针对移动设备的屏幕尺寸和功耗特性进行了精简优化。

五、关机状态行为

5.1 电源选项

移动设备在关机时默认采用“保持在线”模式（基于RCRTOS始终在线的特性），以维持CNS注册和Core Cloud同步能力：

模式	RCRTOS状态	VCRTOS状态	说明
保持在线（默认）	始终运行	最低功耗运行	维持CNS注册、Core Cloud增量同步
完全离线	仍存在于硬件中	停止	设备完全断电
移动设备与标准Core RTOS的关机态行为一致（详见 DOC-2026-055），保留增量同步与中继能力，但功耗目标更低。

5.2 保持在线模式

设备选择“保持在线”关机后：

RCRTOS维持运行（始终在线）
VCRTOS以最低功耗模式运行
保留最小硬件：最低功耗核心、极小内存、存储控制器、网络接口、电源管理单元
功耗目标：< 0.1W（移动设备）

5.3 完全离线模式

设备选择“完全离线”关机后：

VCRTOS停止运行
所有硬件完全断电
下次开机时，RCRTOS首先激活，验证完整性，然后加载VCRTOS
六、与 CNS 的协同（移动版）

6.1 注册模式

模式	说明	适用场景
常驻注册	开机后立即向 CNS 注册，维持在线状态	CorePhone（全天候）
按需注册	仅在需要算力或存储服务时注册	CoreWatch / CoreGlass
离线模式	不注册 CNS，独立运行	无网络环境、IoT设备
6.2 算力消费

移动设备作为 CNS 算力网络的消费者：

设备向 CNS 注册可用算力（或请求算力）
用户发起需要大量计算的任务（AI推理、渲染等）
CNS 将任务调度到 CNBP 或其他算力提供者
结果返回设备
6.3 低功耗通信

使用 CLS 精简协议（低功耗蓝牙 / UWB 版本）
仅在需要时建立连接
支持 异步任务模式：任务提交后，设备可进入休眠，任务完成后通过通知唤醒
七、功耗目标

设备类型	典型功耗	Core RTOS Mobile 贡献	说明
CorePhone	5-15W	<0.5W	后台运行，极低开销
CoreWatch	0.5-2W	<0.1W	超低功耗模式
CoreGlass	1-5W	<0.2W	低延迟唤醒
IoT设备	<1W	<0.05W	极简模式
八、使用场景

8.1 CorePhone

开机 → Core RTOS Mobile 启动（<50ms）
向 CNS 注册设备身份与可用算力
加载默认 OS（Android 或 CoreOS Lite）
后台持续运行，处理安全事件与更新
需要重负载时，通过 CNS 调度 CNBP 算力
8.2 CoreWatch

开机 → Core RTOS Mobile 启动（<30ms）
不主动向 CNS 注册（仅在需要时连接）
加载 CoreOS Lite 超轻量版
后台监控健康数据，异常时触发报警
8.3 CoreGlass

开机 → Core RTOS Mobile 启动（<20ms）
向 CNS 注册（低延迟模式）
加载 CoreOS Lite AR 版
持续接收 CNS 的算力结果（实时渲染辅助）

    ├── CLS精简协议栈（低功耗蓝牙/UWB）
    └── 低功耗自适应验证 + 闲置/夜间更新
