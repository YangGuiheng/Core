一、概述
1.1 定义
CLSDN（Core Link Server Devices Network）是基于CLS协议构建的多层级存储网络组网规范，定义了如何将CNBP、WLCLS和WLHBM组织成一个可扩展、可管理的树状存储网络。

1.2 核心定位
属性	说明
全称	Core Link Server Devices Network
类型	组网协议规范（非硬件）
基础协议	CLS（CoreLink Server）
组成节点	CNBP、WLCLS、WLHBM
最大层级	4层（星型级联）
连接介质	光信号 / 光纤 / STP/UTP双绞线
1.3 解决的问题
CLS协议定义了两个CLS设备之间的点对点通信方式（单跳），但未定义多个设备如何组织成网络（多点拓扑）。CLSDN填补了这一空白：

当有1台CNBP、1台WLCLS、1个WLHBM时，CLS协议已足够

当有1台CNBP、多台WLCLS、数十个WLHBM时，需要CLSDN规范其组网方式

1.4 核心价值
无线存储网络：所有存储节点通过毫米波连接，无需数据线缆

可扩展：通过级联WLCLS，最多可连接343个WLHBM存储节点

海量存储：标准配置下（2TB/WLHBM），单台CNBP可管理686TB存储池

可移动：整机柜搬迁只需一根高压供电线，无需重新布线

二、协议层级关系
2.1 CL与CLS的关系
协议	层级	作用
CL（CoreLink）	基础协议	Core生态所有设备互联的基础通信协议
CLS（CoreLink Server）	协议实现	CL协议在高速服务器互联场景下的具体实现
CLSDN	组网规范	基于CLS协议，定义多设备组网拓扑和层级规则

三、拓扑结构
3.1 默认拓扑：星型
CLSDN默认采用星型拓扑，由一个中心节点连接多个外围节点，结构简洁、易于管理。

text
                    ┌─────────┐
                    │  核心   │
                    │  节点   │
                    └────┬────┘
           ┌─────────────┼─────────────┐
           │             │             │
           ▼             ▼             ▼
      ┌─────────┐  ┌─────────┐  ┌─────────┐
      │ 节点 1  │  │ 节点 2  │  │ 节点 N  │
      └─────────┘  └─────────┘  └─────────┘
3.2 可选拓扑：环形
节点之间形成闭合环路，数据可沿两个方向传输，提供链路冗余。

text
        ┌─────────┐         ┌─────────┐
        │ 节点 1  │─────────│ 节点 2  │
        └────┬────┘         └────┬────┘
             │                   │
             │                   │
        ┌────┴────┐         ┌────┴────┐
        │ 节点 4  │─────────│ 节点 3  │
        └─────────┘         └─────────┘
3.3 可选拓扑：全网状
所有节点之间两两直连，提供最高的冗余性和带宽可用性。

text
        ┌─────────┐         ┌─────────┐
        │ 节点 1  │─────────│ 节点 2  │
        │         │─────────│         │
        └────┬────┘         └────┬────┘
             │                   │
        ┌────┴────┐         ┌────┴────┐
        │ 节点 3  │─────────│ 节点 4  │
        └─────────┘         └─────────┘
3.4 拓扑对比
拓扑	优点	缺点	适用场景
星型	结构简单、易管理、延迟可预测	中心节点单点故障	默认推荐，大多数部署场景
环形	链路冗余、双路径传输	配置复杂	高可用性要求
全网状	极致冗余、带宽充裕	连接数爆炸增长	核心骨干网、容灾场景
四、层级结构
4.1 四层层级定义
CLSDN支持最多四层星型级联，每层由上一级的WLCLS作为中心节点，向下连接多个WLCLS或WLHBM：

层级	角色	说明
第1层（顶层）	CNBP / CLS主控节点	网络的接入点和控制中心
第2层	WLCLS（一级中枢）	直接连接CNBP，管理下层WLCLS或WLHBM
第3层	WLCLS（二级中枢）	由一级WLCLS管理，可继续向下级联或挂载WLHBM
第4层（底层）	WLCLS（三级中枢）/ WLHBM	最底层，全部挂载WLHBM，不继续向下扩展
4.2 端口分配规则
每个WLCLS拥有8个毫米波端口：

端口用途	数量	说明
上行端口（级联）	1个	与上级WLCLS或CNBP通信
下行端口（自由分配）	7个	可连接WLHBM或下一级WLCLS

4.4 层级规则
规则	说明
最大层级	4层（超过4层将导致延迟累积超出CLS协议设计范围）
级联方式	每台WLCLS使用1个端口连接上级，7个端口自由分配
底层存储	第4层节点全部挂载WLHBM，不继续级联
五、最大组网规模
5.1 节点数量计算
按4层满配、每层使用全部7个下行端口进行级联：

层级	设备类型	数量	计算方式
第1层	CNBP	1	根节点
第2层	WLCLS（一级）	1	1 × 1
第3层	WLCLS（二级）	7	1 × 7
第4层	WLCLS（三级）	49	7 × 7
第4层挂载	WLHBM	343	49 × 7
5.2 总存储容量
基于标准配置（每个WLHBM = 2TB）：

配置规模	WLHBM数量	总容量
最小配置（1 WLCLS + 1 WLHBM）	1	2TB
单WLCLS满配（1 WLCLS + 7 WLHBM）	7	14TB
二级级联满配（1+7 WLCLS，全部挂WLHBM）	49	98TB
四级级联满配（最大组网）	343	686TB
5.3 节点间带宽
通信段	介质	带宽
CNBP ↔ WLCLS（一级）	光信号	100GB/s
WLCLS（一级）↔ WLCLS（二级）	毫米波	50GB/s
WLCLS（二级）↔ WLCLS（三级）	毫米波	50GB/s
WLCLS（三级）↔ WLHBM	毫米波	50GB/s
5.4 延迟累积
访问路径	跳数	典型延迟
CNBP ↔ 一级WLCLS	1跳	<10μs
CNBP ↔ 二级WLCLS	2跳	<20μs
CNBP ↔ 三级WLCLS	3跳	<30μs
CNBP ↔ WLHBM（最远路径）	4跳	<40μs
六、连接介质
6.1 介质规格
介质	适用场景	速率	特点
光信号（自由空间）	WLCLS与CNBP/CLS之间	100GB/s	无物理线缆，需视距
光纤	固定部署场景	100GB/s	高可靠性，长距离
STP/UTP双绞线	短距离铜缆连接	受线缆规格限制	兼容现有基础设施
6.2 介质选择建议
部署场景	推荐介质	原因
移动集群、临时部署	光信号（自由空间）	无需布线，快速组网
固定机房、数据中心	光纤	稳定可靠，抗干扰
兼容旧有基础设施	STP/UTP	利用现有布线系统

7.3 存储可见性
访问方式	说明
本地访问	CNBP访问自己的CLSDN存储池（686TB）
集群访问	其他CNBP通过CL协议访问该CNBP的存储池
全局存储池	CL集群将所有CNBP的CLSDN存储池统一挂载，形成共享存储空间
八、典型部署场景
8.1 单机无线存储扩展
配置：1台CNBP + 1台WLCLS + 1-7个WLHBM

连接方式：CNBP通过光信号/毫米波连接WLCLS，WLCLS通过毫米波管理WLHBM

效果：CNBP获得不占用有线接口的无线存储扩展，容量2TB-14TB

8.2 桌面存储中枢
配置：1台CNBP + 1台WLCLS + 7个WLHBM（满配）

连接方式：WLCLS放置于桌面，CNBP进入WorkStation状态时自动建立连接

效果：CNBP自动获得14TB无线存储池，无需任何操作

8.3 海量存储节点
配置：1台CNBP + 4层级联CLSDN网络 + 343个WLHBM

连接方式：CNBP通过光信号连接一级WLCLS，逐级下联

效果：单台CNBP管理686TB存储池，相当于一个移动数据中心

8.4 集群共享存储
配置：5台CNBP组成CL集群，每台挂载各自的CLSDN存储网络

连接方式：CNBP之间通过CL协议互联，各自通过CLSDN连接存储池

效果：总存储池达3.43PB，集群中任意节点均可访问全部存储空间

8.5 移动存储机柜
配置：1台CNBP + 完整CLSDN网络（WLCLS + WLHBM），置于标准机柜中

连接方式：内部无线连接，外部仅需一根高压供电线

效果：整机柜搬迁只需拔插一根电源线，无需重新连接任何数据线
